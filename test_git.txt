git add filename == 기록할 파일을 골라냄 //작업폴더에서 staging area로 가져옴 == 스테이징함
git commit -m "수정설명" == 저장할 내용과 함께 저장 //staging area에서 repository로 저장 == 저장함

보다시피 main branch에서 추가한 설명이 여기에는 없다
branch를 만든 당시에는 완벽히 같은 복제본이었으나 그이후의 커밋은
각기 독립적으로 작용함

branch 끼리 합치려면(merge) 기준이 되는 브랜치로 이종해서  git merge 합칠브랜치이름
    문제는 두 브랜치 간에 똑같은 파일의 똑같은 위치를 다른 수정이 개입되어 있을시 충돌이 발생함
    때문에 같은 파일 같은 위치에 서로 다른 코드가 입력되면 안됨
